Description of : tables

When users are creating content online, one of the common ways to present data or content is by using a table.
We are used to seeing tables in our daily life, for example every time you've used Excel / Google Docs for a task or
when you are buying a new phone and tries to select which paymentplan you would like to choose it is most often written in a table.

Example:

Tables are divided in two different categories of content, you have the tablerow which defines the amount of rows used
in the table and is declared by a <tr>-tag. The second tag which is the tabledata-tag is declared by a <td>-tag,
which adds the data (i.e content) to the related (parent) tablerow-tag (tr).

Now when you know what the rows and data comes from, I'll briefly introduce you to two other tags. The th-tag and caption-tag.
You might know what the rows / columns represent, but if you show your table to another person it might not be as obvious.
by using a <th>-tag you get a bold text which is stating the name of that certain column, for example if you want to
in column A list on each row the names of players in a football team, and column B list the amount of goals they've made this season.
And as far as we know, this table could be absolutely useless since the value of a player (i.e Zlatan) and a random number
might not mean anything if you do not know the table is just a representation of the amount of goals per player this season.
One way to tell the user what the table is all about, the <caption>-tag is a legit way to do it.
